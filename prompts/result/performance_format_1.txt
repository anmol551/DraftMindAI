Write the Results chapter for a research dissertation.
All plots listed in the Result Plot Summary MUST be covered — do not skip any plot.

{figure_rule}

## Chapter Introduction
Begin with a {wc_intro}-word summary of the pipeline followed in the code.
Then present the complete results table with a suitable table name.
Follow the table with a {wc_table_explain}-word explanation of key findings from the table only.

## Structure — Per-Model Subheadings

Create one subheading per model found in the results data. For each model:
- Present every chart belonging to that model (Confusion Matrix, Training & Validation Loss,
  Per-Class Metrics, ROC Curves, or any other chart listed for that model).
- After each chart's figure placeholder, write a discussion paragraph covering metric values,
  patterns, and interpretation.
- Target approximately {wc_per_model} words per model subheading (distributed across its charts).
- Do not combine charts from different models.

## Numbering Rule
All subheadings MUST be numbered starting from 5.1. Use the format "5.X. Heading Name".
- 5.1. to 5.N. → One per model (e.g., 5.1. InceptionV3, 5.2. EfficientNet-B0, ...)
- 5.N+1. Models Comparison
- 5.N+2. WebApp Testing
- 5.N+3. Comparison with Benchmark
- 5.N+4. Novelty
- 5.N+5. Significance and Limitations
Do not use ### or any markdown heading symbols. Write headings as plain numbered text.

After all individual model subheadings, add the following sections:

### Models Comparison
- Present each comparative chart (accuracy comparison, precision/recall/F1 comparison,
  training time comparison, PR curve comparison, or any other comparative chart in the data).
- After each chart, write a discussion paragraph.
- Target approximately {wc_comparison} words.

### WebApp Testing
- Show {wc_webapp_screenshots} UI screenshots as figure placeholders.
- Write a collective discussion of test results in approximately {wc_webapp} words.
- Mention specific test inputs, outputs, and whether predictions matched expectations.

### Comparison with Benchmark
- Compare the pipeline and results of the benchmark paper with this research.
- Approximately {wc_benchmark} words. Do not use parentheses for metric values.

### Novelty
- Rewrite the provided novelty content in the context of the results achieved.
- Approximately {wc_novelty} words. No figure placeholders.

### Significance and Limitations
- Paragraph 1 ({wc_sig_obj} words): How this research fulfils the research objectives.
- Paragraph 2 ({wc_sig_limit} words): Limitations of this research and areas for improvement.
