## Structure — Per-Chart-Type Subheadings with Individual Discussion

## Numbering Rule
All subheadings MUST be numbered starting from 5.1. Use the format "5.X. Heading Name".
- 5.1. Confusion Matrix
- 5.2. ROC Curves
- 5.3. Training and Validation Loss
- 5.4. (any additional chart types, numbered sequentially)
- 5.N. Models Comparison
- 5.N+1. WebApp Testing
- 5.N+2. Comparison with Benchmark
- 5.N+3. Novelty
- 5.N+4. Significance and Limitations
Do not use ### or any markdown heading symbols. Write headings as plain numbered text.

Group charts by type across all models. For each chart type, create one subheading:

### Confusion Matrix
- Present the Confusion Matrix chart for each model, one after another.
- After EACH model's chart, write a separate discussion paragraph covering that specific
  model's true positives, false positives, misclassifications, and patterns.
- Target approximately {wc_per_chart_type} words total for this subheading.

### ROC Curves
- Present the ROC Curve chart for each model, one after another.
- After EACH model's chart, write a separate discussion paragraph covering AUC values,
  class-wise separation, and discrimination ability.
- Target approximately {wc_per_chart_type} words total.

### Training and Validation Loss
- Present the Training & Validation Loss chart for each model, one after another.
- After EACH model's chart, write a separate discussion paragraph covering convergence,
  overfitting/underfitting, and generalisation behaviour.
- Target approximately {wc_per_chart_type} words total.

If additional per-model chart types exist in the data (e.g., Per-Class Precision/Recall/F1,
PR Curves), create a subheading for each and follow the same pattern.

### Models Comparison
- Present each comparative chart (accuracy comparison, precision/recall/F1 comparison,
  training time comparison, PR curve comparison, or any other comparative chart).
- After each chart, write a discussion paragraph.
- Target approximately {wc_comparison} words.

### WebApp Testing
- Show {wc_webapp_screenshots} UI screenshots as figure placeholders.
- Write a collective discussion of test results in approximately {wc_webapp} words.
- Mention specific test inputs, outputs, and prediction accuracy.

### Comparison with Benchmark
- Compare the pipeline and results of the benchmark paper with this research.
- Approximately {wc_benchmark} words. Do not use parentheses for metric values.

### Novelty
- Rewrite the provided novelty content in the context of the results achieved.
- Approximately {wc_novelty} words. No figure placeholders.

### Significance and Limitations
- Paragraph 1 ({wc_sig_obj} words): How this research fulfils the research objectives.
- Paragraph 2 ({wc_sig_limit} words): Limitations of this research and areas for improvement.
