3.1 Research Design

3.1.1 Study Overview  
The research design adopts a supervised machine learning method to sort resumes into job categories based on text content analysis. It involves training algorithms on a dataset of labelled resumes to spot key patterns and make predictions. For instance the approach uses a collection of 2484 resumes from a public source covering various categories to build reliable models. This method suits the project well as it processes text data efficiently and boosts accuracy in sorting tasks. By automating resume checks it reduces manual effort in hiring and manages uneven data distributions effectively helping to handle real scenarios.

3.1.2 Experimental Setup  
The experimental setup begins with data preparation including loading a resume dataset into a framework for analysis then cleaning and dividing it into training and testing parts. It applies text handling steps such as expanding abbreviations removing unwanted elements and simplifying words to create useful features. Models like Naive Bayes bidirectional LSTM and DistilBERT receive these features for training with an 80% training split to ensure balanced representation. This process works for the project by adapting to different resume styles and overcoming sorting issues leading to improved outcomes in a practical tool for classification. The setup also includes imbalance fixes such as weighting classes to enhance performance across categories.

3.2 Data Details

3.2.1 Dataset Source

The dataset consists of resumes drawn from Kaggle's snehaanbhawal collection offering 2484 entries in four columns accessed via KaggleHub for analysis. Kaggle serves as a popular platform for sharing datasets that cover practical applications in fields like job classification making it a straightforward choice for this work. This collection includes diverse resume samples spanning various job roles which provide a solid foundation for examining text patterns and categories. By selecting this source researchers gain exposure to authentic data reflecting common career profiles thus aiding in the development of more accurate classification systems. The dataset's open access means it reaches a wide audience including students exploring machine learning projects.

The decision to use this dataset arises from its ease of access and the broad range of examples present which enhance training processes for categorising jobs. Such real-world data supports better performance by including varied formats and content that mirror actual employment documents.

3.2.2 Data Structure

The data structure features 2484 resumes arranged in four columns containing textual content and category labels which enable easy processing and extraction of key features. This setup includes details like resume text and job categories allowing for quick handling of information during analysis steps. For instance the columns help in identifying patterns such as word frequencies or category trends that inform model building. 

This arrangement benefits the project by simplifying text management and ensuring that data flows smoothly into classification tasks. It offers a clear layout that reduces complexity in early stages and allows for effective comparison across different resumes helping to build a reliable framework for outcomes. The structured format also accommodates potential expansions like adding more features without major changes.

Data preprocessing transforms raw resume data into a clean format for analysis. This stage ensures the resume dataset becomes suitable for machine learning by handling inconsistencies and preparing features. The project uses a structured approach on a dataset of 2 484 resumes to enhance data quality and support classification tasks. Such preparation reduces errors and streamlines subsequent steps.

3.3.1 Feature Elimination

Feature elimination discards irrelevant columns to simplify the dataset. This action removes items like ID and Resumehtml which do not contribute to analysis. By focusing on key resume elements such as text content the process aids in building more effective models for job category detection.

3.3.2 Text Expansion

Text expansion changes shortened forms into full words to maintain consistency. This step applies tools to convert abbreviations ensuring that resume language appears uniform. As a result patterns in text become clearer which supports better identification of job categories.

3.3.3 Text Cleaning

Text cleaning removes unwanted parts to standardise content. This includes actions like converting text to lowercase and eliminating URLs emails phones and special characters. Through these efforts the resumes show clearer patterns that help models learn from the data more effectively.

3.3.4 Token Processing

Token processing divides text into separate words and removes frequent non-specific ones. This method keeps important terms related to resumes while cutting out common words. It creates a refined dataset that highlights relevant features for improved classification outcomes.

3.3.5 Word Normalization

Word normalization converts words to their base forms to handle variations. This technique applies lemmatization to simplify language across resumes. Such changes lead to consistent features that allow models to perform better in predicting job categories.

3.3.6 Imbalance Handling

Imbalance handling addresses uneven distribution of classes in the dataset. This correction ensures each category gets proper representation to avoid bias in models. The adjustment promotes fairer results and boosts performance in classification tasks. For example methods like weighting help balance classes and improve accuracy to around 83.50%.

3.4 Feature Engineering

3.4.1 Text Vectorization  
Text vectorization transforms raw text into numerical data for machine learning. TF-IDF represents words by calculating their frequency in a document against their rarity across all documents.  

This approach was chosen for the project as it converts resume text into meaningful features that help models detect important patterns in job descriptions and skills improving classification accuracy.

3.4.2 Category Encoding  
Category encoding changes non-numerical labels into numbers that algorithms can use. Label encoding assigns a distinct integer to each category in the dataset.  

The method was selected because it simplifies the 24 job categories into numerical form essential for training models and ensuring they process the data without errors.

3.4.3 Entity Recognition  
Entity recognition locates and categorises specific items in text such as skills or education terms within resumes. It uses tools to identify these elements and count their occurrences.  

The process was adopted to pull out key details from resumes adding custom features like skill counts that enhance model performance in categorising documents based on content.

3.6 Performance Evaluation

3.6.1 Accuracy Measure
Accuracy indicates the share of correct predictions out of all predictions made by the model. It calculates the total number of correct classifications divided by the total instances.
In this resume classification project accuracy helped gauge how well the model identified job categories overall helping to compare different models like DistilBERT against baselines.

3.6.2 Precision Measure
Precision reflects the proportion of true positive predictions among all positive predictions. It counts correct positive results divided by the sum of correct and incorrect positives.
For this study precision was selected to evaluate the model's ability to avoid false positives in resume categorisation ensuring reliable predictions for imbalanced job classes.

3.6.3 Recall Measure
Recall means the proportion of true positive predictions out of all actual positive instances. It divides true positives by the total of true positives and false negatives.
This project used recall to assess how effectively the model captured relevant resumes in each category addressing the need to detect minority classes in the dataset.

3.6.4 F1 Measure
F1 measure combines precision and recall into a single score using the harmonic mean. It balances both metrics to provide a comprehensive view of model performance.
In this work F1 was chosen because it offered a balanced assessment of precision and recall vital for evaluating resume models with uneven class distributions.

3.6.5 Mean Measure
Mean measure here refers to the average of evaluation scores across classes. It computes the arithmetic average of metrics like F1 for all categories.
The project applied mean measure to summarise performance across 24 job categories allowing a fair comparison of models on imbalanced data.

3.6.6 Matrix Analysis
Matrix analysis involves examining the confusion matrix which shows correct and incorrect predictions in a grid. It displays true positives false positives and other errors.
This analysis was used in the study to identify specific misclassifications in resume data helping to refine models and understand category-specific weaknesses.

3.5 Models

This part describes the machine learning models chosen to sort resumes into job categories. The models process pre-treated text data to spot patterns and make predictions based on word patterns.

3.5.1 Multinomial Naive Bayes

Multinomial Naive Bayes serves as a basic classification tool that follows Bayes' theorem and treats features as separate units. It assesses the probability of a document belonging to a category by looking at word frequencies from the training data. This approach uses a multinomial setup to handle word counts which suits text-based tasks like resume sorting. For example the model counts how often words appear in different job types and uses those counts to classify new resumes.

The model offers a straightforward design that lets it work fast on large text sets. In resume work it matches well with clear job groups because it delivers quick results and steady performance. Researchers often use it for text issues where word totals matter most as it provides an easy way to start analysis. One drawback is that it ignores word order so it might miss some context in detailed resumes. Still its low demand for computing power makes it ideal for initial tests. From the setup it works with TF-IDF features and an alpha value of 0.1 to smooth counts and improve stability. This adjustment helps the model deal with imbalanced data by giving more weight to less common categories ensuring better fairness across job types.

In practice the model builds on earlier ideas from probability theory and adapts them for modern text needs. Compared to decision trees it offers faster training times which benefits projects with tight schedules. Its ability to handle high-dimensional data like resume text without much hassle adds to its appeal in educational settings.

3.5.2 Bidirectional LSTM

Bidirectional LSTM is a recurrent neural network design that reviews text sequences from both directions to capture full context. It relies on memory cells to hold onto information over long parts of the text helping it link ideas that appear far apart. The structure includes gates that manage which data to retain or forget allowing it to adapt to different text sizes.

For resume classification this model captures word flow which is key for grasping job context. It can for instance trace how a skill connects to a role across multiple lines leading to more precise predictions. The two-way flow lets it pick up on hidden patterns that simpler models overlook. Based on the setup it uses an embedding layer followed by BiLSTM layers with 128 hidden units and dropout to prevent overfitting during training.

The model draws from neural network advances and applies them to sequence problems making it effective for tasks like text analysis. In contrast to basic Naive Bayes it accounts for sequence which boosts accuracy on complex data. However it needs more processing time and resources which could slow down experiments on standard computers. Despite this its strong results on sequential text make it a good option for deeper resume insights. Training involves 10 epochs with the Adam optimiser and a learning rate scheduler to refine performance over time.

3.5.3 DistilBERT

DistilBERT acts as a compact version of the BERT model using transformer layers to examine text for classification. It employs attention to focus on important sentence parts and understand word links. The model takes insights from its larger counterpart but reduces size for quicker operation.

In resume classification DistilBERT achieves high accuracy with fast processing which fits applications that require instant feedback. It starts with pre-trained weights and adjusts them on specific data like job texts to enhance outcomes. For example it handles complex language by learning from patterns in training sets leading to reliable sorting.

The model stems from recent developments in natural language processing and offers a balance of speed and effectiveness. Unlike traditional methods it captures word relationships without heavy computation making it suitable for varied text lengths. In the process it uses a sequence length of 256 and trains for 10 epochs with the AdamW optimiser to fine-tune parameters. This setup allows it to perform well on imbalanced data through weighted loss functions improving its predictions for minority categories. Overall it provides a practical choice for text tasks in research projects.
