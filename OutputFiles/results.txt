1. RESULTS
----------------------------------------
The code pipeline begins with data loading using pandas and numpy, followed by exploratory data analysis with seaborn and matplotlib. Preprocessing involves nltk for tokenisation and lemmatisation, feature engineering with sklearn TfidfVectorizer, and model training using torch for Bi-LSTM and transformers for DistilBERT. Evaluation occurs via sklearn metrics, with results saved using joblib and deployed in a Flask web app.

The table shows DistilBERT variants as top performers, with accuracy reaching 84% and G-Mean up to 78%. The results highlight model strengths in multi-class classification, with DistilBERT excelling overall.

5.6 Comparison with Benchmark
This research's DistilBERT model achieves 84% accuracy, surpassing the benchmark paper by Jalili et al. (2024) which reported 72.4% with BiLSTM on a similar dataset. The current pipeline incorporates advanced imbalance handling, leading to better G-Mean scores, unlike the benchmark's focus on bidirectional processing alone.

5.7 Novelty
The research offers a streamlined pipeline that tests multiple models side by side, including MNB, Bi-LSTM and DistilBERT, under the same setup to pinpoint true performance differences. It uses a custom G-Mean measure to spotlight issues with minority classes, showing how DistilBERT reaches 77.8% G-Mean after corrections. The approach applies various imbalance fixes like weighting and threshold adjustments, revealing their benefits for DistilBERT but not Bi-LSTM. Additionally, it integrates NER for skills and roles, turning raw text into usable data. This full pipeline, ending in a web app, provides a ready tool for real-world use, extending beyond simple classification.

5.8 Significance and Limitations
The research meets objectives by creating a predictive framework that compares models on financial-like datasets, achieving high accuracy with DistilBERT and integrating explanations via NER. It examines data preparation effects, as seen in preprocessing steps, and deploys a web app for practical decision support, fulfilling the goal of usable tools.

Limitations include the dataset's class imbalance, which affects minority category performance despite corrections. Future work could expand the dataset and explore more advanced imbalance techniques to enhance accuracy across all classes.
