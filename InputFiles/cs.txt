The codebase implements a complete end-to-end automated skin lesion classification pipeline using the HAM10000 dermatoscopic dataset. Starting from raw JPEG images and metadata, the system performs extensive EDA to characterise class imbalance, resolution distributions, RGB channel properties, and brightness profiles across seven lesion categories. A structured preprocessing pipeline applies DullRazor-based hair removal, per-channel colour normalisation, and CLAHE contrast enhancement before training, ensuring that spurious image artefacts do not bias model learning. Lesion-level train-validation-test splitting prevents patient-level data leakage, and a WeightedRandomSampler compensates for severe class imbalance (nv dominates with 6,705 samples versus df with 115). Four transfer learning architectures — InceptionV3, EfficientNet-B0, Xception, and ConvNeXt-Tiny — are trained under identical protocols using pretrained ImageNet weights, frozen backbones, custom classification heads, Adam optimiser, CrossEntropyLoss, and StepLR scheduling. Each model is evaluated with confusion matrices, per-class precision/recall/F1 bar charts, training/validation loss curves, and multi-class ROC curves. ConvNeXt-Tiny achieves the best results across all metrics (accuracy 77.34%, weighted F1 78.29%, micro AUC 0.9665, melanoma recall 58.58%), outperforming InceptionV3 (66.58%), EfficientNet-B0 (70.48%), and Xception (74.17%). The trained ConvNeXt-Tiny model is deployed as a Flask web application featuring image upload, real-time inference, per-class probability display, and a high-risk melanoma alert when predicted probability exceeds 0.60.
