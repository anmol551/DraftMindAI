1. **Environment Setup:** Import core libraries (NumPy, Pandas, PyTorch, torchvision, timm, OpenCV, scikit-learn, Matplotlib, Seaborn, tqdm). Set global seeds (SEED=42) for reproducibility. Detect device (CUDA GPU or CPU). 2. **Data Loading:** Load HAM10000 metadata CSV. Map image IDs to file paths across HAM10000_images_part_1 and part_2 directories. Apply class name mapping for seven diagnostic categories. 3. **Exploratory Data Analysis (EDA):** Analyse class distribution, binary melanoma vs. non-melanoma split, lesion-wise image counts, image resolutions (scatter plots), RGB channel histograms, per-class brightness, colour distributions, and edge detection demonstrations using Canny filters. 4. **Preprocessing Pipeline:** Apply DullRazor algorithm (morphological blackhat + inpainting) for hair removal. Apply channel-level colour normalisation (per-channel mean normalisation). Apply CLAHE (Contrast-Limited Adaptive Histogram Equalisation on LAB L-channel). Resize all images to 224×224 and normalise to [0,1]. Visualise preprocessing effects per class. 5. **Train-Validation-Test Split:** Split by unique lesion_id (70% train, 15% val, 15% test) to prevent patient-level data leakage. Confirm class distribution in each split. 6. **Data Augmentation and Sampling:** Training transforms: RandomHorizontalFlip, RandomVerticalFlip, RandomRotation(20°), ColorJitter, ToTensor, ImageNet normalisation. Val/Test transforms: Resize, ToTensor, ImageNet normalisation. Compute class weights using inverse frequency. Apply PyTorch WeightedRandomSampler to address class imbalance. 7. **Dataset and DataLoader Creation:** Implement custom SkinDataset (PyTorch Dataset) loading images on-the-fly with preprocessing and transforms. Create DataLoaders: batch_size=32, sampler for train, num_workers=2, pin_memory=True. 8. **Model 1 — InceptionV3:** Load pretrained InceptionV3 (ImageNet weights). Freeze backbone layers. Replace final FC layer with custom head (7-class output). Train with CrossEntropyLoss, Adam (lr=0.001, weight_decay=1e-4), StepLR scheduler. Save best checkpoint. Evaluate: accuracy 66.58%, weighted F1 69.18%, micro AUC 0.941. Generate confusion matrix, per-class precision/recall/F1 bar charts, training/validation loss curves, ROC curves. 9. **Model 2 — EfficientNet-B0:** Same pipeline with EfficientNetB0 backbone. Custom classification head replaces top FC. Best accuracy 70.48%, weighted F1 72.56%, micro AUC 0.955. Same evaluation plots generated. 10. **Model 3 — Xception:** Implemented via timm library. Frozen backbone with custom head. Best accuracy 74.17%, weighted F1 75.47%, micro AUC ~0.962. Same evaluation suite. 11. **Model 4 — ConvNeXt-Tiny:** Load pretrained ConvNeXt-Tiny. Replace classifier with custom Sequential head: Flatten → LayerNorm(768) → Linear(768→512) → ReLU → Dropout(0.5) → Linear(512→7). Train with same protocol. Best accuracy 77.34%, weighted F1 78.29%, micro AUC 0.9665. 12. **Model Comparison:** Compare all four models on Accuracy, Precision, Recall, F1-Score, ROC-AUC (Micro) using bar charts, grouped metric comparison charts, Precision-Recall curves (AP scores), training time dot plot, and Cohen's Kappa. ConvNeXt-Tiny ranked best on all metrics. 13. **Results Export:** Save comparison DataFrame to CSV. Display styled performance table with green highlighting on best values. 14. **Web Application Deployment:** Flask app (app.py) loads saved ConvNeXt-Tiny checkpoint at startup. User uploads a dermoscopic image via HTML form. Image preprocessed (OpenCV BGR→RGB, Resize 224×224, ToTensor, ImageNet normalise). Inference with torch.no_grad(). Softmax probabilities computed. Predicted class displayed with all class probabilities. High-risk melanoma alert triggered if mel probability ≥ 0.60.
