The web application is built using the Flask framework and serves as a clinical-facing dermatoscopic image classification interface. Key dependencies include Flask, PyTorch, torchvision, OpenCV, and NumPy. User input is collected via an HTML file-upload form on the index route. The uploaded image is saved to the static/uploads directory, preprocessed using OpenCV (BGR to RGB conversion, resize to 224×224, ToTensor, ImageNet normalisation), and passed to the loaded ConvNeXt-Tiny model for inference. The model — a custom ConvNeXtTinyClassifier with a Sequential head (Flatten → LayerNorm(768) → Linear(768→512) → ReLU → Dropout(0.5) → Linear(512→7)) — is loaded once at startup from a saved .pth checkpoint. Softmax probabilities are computed for all seven lesion classes. The predicted class label is mapped to its full medical name (e.g., 'mel' → 'Melanoma') and displayed alongside all class probability scores formatted to four decimal places. A high-risk melanoma alert (risk_flag=True) is triggered if the predicted class is 'mel' and its confidence exceeds 0.60. Results are rendered back to the user via the index.html Jinja2 template.
